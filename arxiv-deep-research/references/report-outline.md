# Report outline (default)

Use this structure when producing a deep, survey-style writeup.

## 1) TL;DR (insights, not summaries)

- 5–10 evidence-backed claims, each with citations.
- Highlight what is *surprising* / *counterintuitive*.

## 2) Scope and methodology

- In scope / out of scope
- Search strategy (queries + categories + time window)
- Inclusion/exclusion criteria
- Limitations (what you might be missing)

## 3) Taxonomy (the map)

Organize the area by:

- tasks / problem settings
- method families
- key design choices (objectives, architectures, data, training tricks)
- evaluation protocols and datasets

## 4) Core papers (annotated)

For each core paper:

- What it introduced
- Why it matters
- Main empirical result(s)
- Limitations

## 5) Comparative analysis

- Where gains come from (and when they disappear)
- Trade-offs: compute, data, robustness, controllability, safety
- Failure modes
- Contradictory evidence and likely causes
- Bias and reliability notes (publication bias, leakage, contamination, cherry-picking)

## 6) Open problems and research directions

Each direction should include:

- the gap
- why it matters
- a concrete experiment / benchmark / ablation to test it

## 7) Appendix: paper list + BibTeX

- papers.md (export table)
- papers.bib
- search_log.md
- evidence-table.md (if used)
